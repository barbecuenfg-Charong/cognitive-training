## 4. 今天推动的核心变化

今天的核心变化不是“多写了几个页面”，而是训练项目的工作重心被重新固定下来。

### 4.1 从结果落库转向训练质量

此前项目的重要里程碑是 33 个 active 训练任务完成 `TrainingResults.saveSession` 接入。今天进一步明确：落库只是基础，不等于训练有效。

下一阶段要看：

1. 题目是否参数化。
2. 难度是否能跟随表现变化。
3. 指标是否能对应认知范式。
4. 错误是否能分型。
5. 反馈是否能指导下一轮训练。

### 4.2 从固定 demo 转向能力边界训练

今天强化的模块大量加入了：

1. adaptive staircase 或动态参数。
2. trial-level 记录。
3. seed / contentVersion / sessionType 等可复盘字段。
4. hit、miss、false alarm、commission、interference cost、calibration 等更贴近理论目标的指标。
5. 下一轮处方或训练反馈。

这意味着训练不再只看总分，而开始看用户在哪类任务条件下出错。

### 4.3 多 agent 模式需要主 agent 闸门

今天也明确了协同模式的修正：

1. 子 agent 不能自由扩散任务。
2. 每个 worker 必须有明确写入范围。
3. 审计 agent 不能抢跑，必须等 worker 完成后再审。
4. 主 agent 不应陷入具体实现细节，但必须负责目标纠偏、写集检查和验收门禁。

这个规则后续会直接影响开发分派方式。
